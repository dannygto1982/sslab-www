第一章　系统概述

1.1 产品简介

SSLAB 实验室综合管理系统是专为教学实验室设计的一体化智能管控平台。系统以 浏览器为操作界面，通过局域网与各类实验室设备通信，实现对教师电源、学生分组供电、高压设备、 升降讲台、照明灯具及环境传感器的集中统一管控，大幅降低实验室安全风险并提升教学效率。

系统后端采用 Python + FastAPI 框架，通过 WebSocket 向前端 推送实时状态，支持多客户端同时在线查看。前端为纯 HTML/CSS/JS 页面，兼容 PC 浏览器和平板设备。

1.2 主要功能

⚡
教学电源管理

远程设置教师低压直流/交流电源的输出电压与限流值，一键开关控制。

🔌
学生分组供电

对 A/B/C/D 四个学生组独立控制供电开关，支持全组同步低压输出。

⚠️
高压电源控制

240V–300V 高压选择及主开关控制，内置 10s 延时保护倒计时。

🖥️
升降台 / 电脑

广播控制全教室升降讲台升降动作，同时管理电脑桌面电源开关。

💡
灯光 / 环境控制

黑板灯、教室灯开关，供水阀、排风扇独立控制，环境数据实时显示。

🌐
网络设备扫描

自动扫描局域网 192.168.0.x 网段，发现并注册在线设备，每 60s 自动刷新。

第二章　安装与启动

2.1 运行环境要求

项目要求
操作系统Windows 10/11（推荐）或 Linux
Python3.10 或以上版本
网络与实验室设备处于同一 192.168.0.x 局域网
浏览器Chrome 90+、Edge 90+、Firefox 88+（支持 WebSocket）
主要依赖库fastapi、uvicorn、websockets、aiofiles

2.2 快速启动步骤

  1. 安装依赖
    在项目根目录执行： 
    pip install -r backend/requirements.txt
  2. 启动后端服务
    执行以下命令，服务将监听 0.0.0.0:1880： 
    python run_backend.py
    启动成功后，控制台输出： 
    Starting Control System Server...
[AutoScan] Initial scan started...
[Server-8887] Listening on ('0.0.0.0', 8887)
  3. 打开前端界面
    在浏览器地址栏输入： 
    http://localhost:1880
    或使用服务器真实 IP（供其他设备访问）： 
    http://192.168.0.x:1880
  4. 确认设备连接
    界面右上角连接状态指示灯显示在线， 各功能模块状态徽章由未连接变为 已连接即表示设备正常。
💡 提示 系统启动后会自动进行一次全网段扫描（约需 5–15 秒）， 随后每 60 秒周期性刷新一次设备列表，无需手动操作。

2.3 仿真模式启动（无实体设备）

在没有实体设备的环境下，可启动内置仿真服务进行功能验证：

  1. 启动虚拟设备服务器（模拟端口 8887 / 1053）： 
    python backend/mock_device_server.py
  2. 启动仿真后端（使用 start_simulation.bat）： 
    .\start_simulation.bat
  3. 或使用 VS Code 任务：按 Ctrl+Shift+B → 选择 Run ALL (Simulation)， 系统将自动启动仿真 GUI、后端服务并打开浏览器。
⚠️ 注意 仿真模式下发送的控制指令不会传达给真实设备，仅用于测试界面交互逻辑。 生产环境中请确保服务器接入实际局域网。

第四章　教学电源管理

4.1 低压教学电源设置

对应设备：192.168.0.211:8888（教师电源，Modbus RTU over TCP）。 此模块控制教师讲台低压电源的输出参数与开关状态。

控件名称
功能说明
实时电压显示
显示电源当前输出电压（从设备轮询或 WebSocket 推送，单位：V）。
设置输出电压
输入目标输出电压（0–30 V，步进 0.1V）。修改后立即下发指令（前提：电源已开启）。
实时电流显示
显示当前输出电流（单位：A）。
设置限制电流
设置电流上限（0–2 A，步进 0.1A）。默认值 2.5 A，设为 0 时系统自动使用默认值。
输出控制开关
控制电源主输出通断（对应系统变量 LowKZ）。开关拨动后立即发送 Modbus 指令。
直流 / 交流 切换
切换输出类型（DC → Slave 0xA1，AC → Slave 0xA2）。切换后若电源已开启会自动重发设定参数。
⚠️ 操作提示 先设置好电压与电流上限，再开启输出控制开关，可避免上电瞬间因参数未设定导致的异常。 调节参数时，若「输出控制」已为开启状态，每次参数变动都会立即生效。

4.2 高压电源设置

对应设备：192.168.0.7:8234（USR-IO808 继电器控制板，Modbus TCP）。

控件名称
功能说明
240V–300V 选择
选择高压输出档位（低档 240V / 高档 300V），对应继电器寄存器 0x0006（HighXZ）。
高压主开关
控制高压主回路通断（HighKZ，寄存器 0x0007）。
开启时有 10 秒安全倒计时，倒计时结束前可取消操作。
大电流电源开关
控制大电流输出回路（HighCurrent，寄存器 0x0000），同样有 10 秒延时保护。
电源插座开关
控制实验台电源插座供电（PowerCZ，寄存器 0x0005）。
🚨 安全警告 高压电源（240V–300V）接通前请务必确认实验台已清场，无师生接触导线。 高压主开关开启后，倒计时期间红色指示灯闪烁，请密切观察实验区域。

4.3 学生电源控制与分组开关

学生电源通过端口 8887 的学生同步服务器推送至各学生设备（Modbus RTU over TCP）， 分组控制则通过继电器控制板（端口 8234）实现。

控件名称系统变量寄存器地址说明
低压电源同步LowXSTB—将教师低压参数同步推送至所有在线学生设备
A 组开关XS_A0x0004控制 A 组学生台供电继电器
B 组开关XS_B0x0003控制 B 组学生台供电继电器
C 组开关XS_C0x0002控制 C 组学生台供电继电器
D 组开关XS_D0x0001控制 D 组学生台供电继电器
💡 提示 学生分组可独立开关，教师可在实验开始前按组别控制供电，灵活安排实验节奏。 分组开关状态即时生效，无延时保护。

5.3 升降台与电脑管理

系统通过广播方式向 192.168.0.100–130 的所有升降台控制器发送指令（端口 1053， Legacy JSON 协议）。状态徽章显示当前在线（响应的）设备数量。

升起 / 落下（Lifting）
开关拨向「开」→ 向上升起（motor_fwd: true）；
拨向「关」→ 向下落下（motor_bwd: true）。
指令以 2 次广播方式发送，确保可靠到达。
电脑开关（Computer）
控制电脑桌电源继电器（device1）。 广播至同一 IP 范围内的所有学生桌控制器。
⚠️ 注意 升降台操作时请提前告知学生远离讲台边缘，避免夹伤。 升降指令广播给所有在线设备，会同步控制所有学生桌，请谨慎操作。

5.4 灯光控制

第六章　网络设备管理

6.1 设备发现与扫描

系统扫描 192.168.0.0/24 网段，对每个 IP 并发探测以下端口，按响应判断设备类型：

端口设备类型设备名称
8887student学生同步设备（亿佰特 NT1-B）
1053control升降台控制器（鸣驹 MJ-E1S1）
8888teacher教师电源（Teacher-Power）
8234device_8234综合继电器控制板（USR-IO808）

扫描结果保存在 backend/config.json 中，格式示例如下：

{
  "devices": [
    { "ip": "192.168.0.7",   "port": 8234, "type": "USR-IO808",      "id": "Device_8234" },
    { "ip": "192.168.0.211", "port": 8888, "type": "TeacherPower",   "id": "Device_8888" },
    { "ip": "192.168.0.12",  "port": 8887, "type": "SyncServer",     "id": "Device_8887" },
    { "ip": "192.168.0.100", "port": 8889, "type": "EnvSensor",      "id": "Device_8889" }
  ]
}

可通过 API 触发手动扫描（详见第七章）。扫描超时时间为 3 秒/端口，全网段扫描约需 5–30 秒。

6.2 设备配置文件

配置文件 backend/config.json 由扫描结果自动生成，也可手动编辑以固定设备 IP。 系统在发送控制指令前会读取此配置文件以确定目标地址。

💡 固定设备 IP 建议在路由器中为各实验室设备绑定静态 IP，避免 DHCP 分配变化导致扫描结果失效。 配置文件中的 IP 地址优先级高于自动扫描结果。
💡 继电器控制板固定地址 USR-IO808（端口 8234）的 IP 固定为 192.168.0.7， 即使扫描未发现也会强制包含在控制目标列表中。

第七章　API 接口参考

系统所有接口基址为 http://<服务器IP>:1880，支持 JSON 格式请求与响应。

7.1 状态查询接口

GET /req 获取当前所有设备状态（完整状态对象）

响应示例：

{
  "LowKZ": false,   // 低压电源开关
  "HighKZ": false,  // 高压电源主开关
  "VFD_Speed": 1,   // 变频器速度档位
  "LowDYSZ": 0,    // 设定电压值 (V)
  "LowDLSZ": 2.0,  // 设定电流值 (A)
  ...
}

7.2 设备控制接口

POST /{domain}/{device_id} 控制指定设备（通用控制端点）

请求体（JSON）：

{ "value": true }          // 布尔值或数值均可
{ "value": 12.5 }          // 数值参数（如电压）

常用控制示例：

请求说明
POST /ctrl/LowKZ {"value": true}开启低压电源输出
POST /ctrl/LowDYSZ {"value": 12.0}设置输出电压为 12V
POST /ctrl/HighKZ {"value": true}开启高压主开关（触发 10s 倒计时）
POST /ctrl/XS_A {"value": false}关闭 A 组学生分组开关
POST /ctrl/Lifting {"value": "up"}升降台上升
POST /ctrl/Lifting {"value": "stop"}升降台停止
POST /ctrl/Computer {"value": true}开启电脑电源
POST /ctrl/BBLampKZ {"value": true}开启黑板灯
POST /ctrl/VFD_Speed {"value": 2}变频器切换至二档

7.3 网络扫描接口

GET /api/scan 触发全网段设备扫描
GET /api/scan?clear=true 清除缓存后重新扫描
GET /api/scan?ports=8234,8888 仅扫描指定端口（逗号分隔）
GET /api/devices 获取当前已记录的设备列表

7.4 WebSocket 接口

WS ws://<服务器IP>:1880/ws 实时状态推送（前端自动连接）

服务器推送消息类型：

// 设备状态更新
{ "type": "state", "data": { "LowKZ": true, ... } }

// 指令执行结果
{ "type": "cmd_result", "ip": "192.168.0.7", "status": "success" }

// 扫描完成通知
{ "type": "scan_complete", "count": 4, "data": [...] }

第八章　设备协议参考

8.1 教师电源（端口 8888）— Modbus RTU over TCP

协议特点：无 MBAP Header，原始 RTU 帧（含 CRC16-Modbus）直接经 TCP 传输。

模式Slave ID寄存器地址含义
DC（直流）0xA10x0003电流上限（单位 0.001A，2500 = 2.5A）
DC（直流）0xA10x0004电压设定（单位 0.01V，850 = 8.50V）
DC（直流）0xA10x0005使能控制（1=ON，0=OFF）
AC（交流）0xA2同上同上

开启帧（DC，12V / 2.5A）：

A1 10 00 03 00 04 08 09 C4 04 B0 00 01 00 00 46 3E
│  │  │─────│  │  │  │──────│──────│──────│──────│──────────
│  │  起始0x0003  4  8 字节  电流   电压    ON   预留  CRC
Slave Func(0x10)  寄存器数

8.2 继电器控制板（端口 8234）— Modbus TCP

协议特点：标准 Modbus TCP，含 MBAP Header（6 字节前缀），功能码 0x06 写单寄存器，无 CRC。

帧字段值说明
Transaction ID00 01递增事务 ID
Protocol ID00 00固定 0x0000（Modbus TCP）
Length00 06后续字节数 6
Unit ID01固定 0x01
Function Code06写单个寄存器
Register Address00 XX见附录寄存器映射表
Register Value00 01/001=ON，0=OFF

示例（开启继电器通道 0，即 HighCurrent）：

00 01 00 00 00 06 01 06 00 00 00 01

8.3 升降台控制器（端口 1053）— Legacy JSON/TCP

协议特点：纯文本 JSON 报文，以 \n 换行符结尾。

指令JSON 内容说明
电脑桌电源开{"device1": true}\n电脑电源继电器接通
电脑桌电源关{"device1": false}\n电脑电源继电器断开
升降台上升{"motor_fwd": true}\n电机正转（上升）
升降台下降{"motor_bwd": true}\n电机反转（下降）
升降台停止{"motor_fwd": false, "motor_bwd": false}\n电机停止

指令以广播方式发送至 192.168.0.100–130 的所有在线控制器，每条指令发送 2 次以提高可靠性。
