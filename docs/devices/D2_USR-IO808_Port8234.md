# D2 — USR-IO808 综合控制器

> **端口：** TCP 8234  
> **固定 IP：** 192.168.0.7  
> **协议：** Modbus TCP（功能码 0x06，写单个寄存器）  
> **连接方式：** 短连接（发完即断）

---

## 一、硬件说明

USR-IO808 是一款 8 路以太网数字量 IO 控制器（有人科技），负责实验室强电回路的通断控制：

| 寄存器地址 | 变量名 | 功能说明 |
|-----------|-------|---------|
| `0x0000` | `HighCurrent` | 高电流控制 |
| `0x0001` | `XS_D` | D 组学生台供电开关 |
| `0x0002` | `XS_C` | C 组学生台供电开关 |
| `0x0003` | `XS_B` | B 组学生台供电开关 |
| `0x0004` | `XS_A` | A 组学生台供电开关 |
| `0x0005` | `PowerCZ` | 电源插座 |
| `0x0006` | `HighXZ` | 高压选择开关 |
| `0x0007` | `HighKZ` | 高压主控开关 |
| `0x0008` | `BBLampKZ` | 白板灯 |
| `0x0009` | `CRLampKZ` | 顶灯 |

## 二、通信协议

### Modbus TCP 帧结构（共 12 字节）

```
┌──────────────────────────────────────────────────────────────────┐
│  MBAP Header (7 bytes)          │  PDU (5 bytes)                │
├──────────┬──────────┬───────────┬───────┬────────┬──────────────┤
│ Trans ID │ Protocol │  Length   │ Unit  │  Func  │  Addr  │ Val │
│  2 bytes │  2 bytes │  2 bytes  │ 1 byte│ 1 byte │ 2 bytes│2 B  │
│  0x0001  │  0x0000  │  0x0006   │ 0x01  │  0x06  │ 0xXXXX │0/1 │
└──────────┴──────────┴───────────┴───────┴────────┴──────────────┘
```

- **功能码：** `0x06`（写单个寄存器）
- **Unit ID：** `0x01`
- **值：** `0x0001` = 开，`0x0000` = 关

### 帧构造（Python）
```python
import struct

def groups_modbus_tcp_cmd(addr: int, val: int) -> bytes:
    trans_id = 0x0001
    proto    = 0x0000
    length   = 0x0006
    unit     = 0x01
    func     = 0x06
    header = struct.pack('>HHHB', trans_id, proto, length, unit)
    pdu    = struct.pack('>BHH', func, addr, val)
    return header + pdu
```

## 三、各通道命令帧（完整字节）

### XS_A 开（0x0004 = 1）
```
00 01  00 00  00 06  01  06  00 04  00 01
```

### XS_A 关（0x0004 = 0）
```
00 01  00 00  00 06  01  06  00 04  00 00
```

### XS_B 开（0x0003 = 1）
```
00 01  00 00  00 06  01  06  00 03  00 01
```

### XS_B 关（0x0003 = 0）
```
00 01  00 00  00 06  01  06  00 03  00 00
```

### XS_C 开（0x0002 = 1）
```
00 01  00 00  00 06  01  06  00 02  00 01
```

### XS_D 开（0x0001 = 1）
```
00 01  00 00  00 06  01  06  00 01  00 01
```

### HighKZ 开（高压主控，0x0007 = 1）
```
00 01  00 00  00 06  01  06  00 07  00 01
```

### HighXZ 开（高压选择，0x0006 = 1）
```
00 01  00 00  00 06  01  06  00 06  00 01
```

### HighCurrent 开（0x0000 = 1）
```
00 01  00 00  00 06  01  06  00 00  00 01
```

### BBLampKZ 开（白板灯，0x0008 = 1）
```
00 01  00 00  00 06  01  06  00 08  00 01
```

### CRLampKZ 开（顶灯，0x0009 = 1）
```
00 01  00 00  00 06  01  06  00 09  00 01
```

### PowerCZ 开（电源插座，0x0005 = 1）
```
00 01  00 00  00 06  01  06  00 05  00 01
```

> 所有关闭命令将最后 2 字节改为 `00 00` 即可。

## 四、API 调用方式

```http
POST /ctrl/{device_id}
Content-Type: application/json

{"value": true}   # 开
{"value": false}  # 关
```

**支持的 device_id 列表：**
`XS_A`、`XS_B`、`XS_C`、`XS_D`、`HighKZ`、`HighXZ`、`HighCurrent`、`BBLampKZ`、`CRLampKZ`、`PowerCZ`

**示例：开启 A 组学生台**
```bash
curl -X POST http://localhost:1880/ctrl/XS_A \
  -H "Content-Type: application/json" \
  -d '{"value": true}'
```

## 五、设备响应

Modbus TCP 标准应答（6 字节），内容为原命令回显，后端不做校验。

## 六、IP 固定说明

IP `192.168.0.7` 已在代码中强制写入，即使扫描未发现该设备也会强制发送：

```python
# 强制包含固定IP
if not found_fixed:
    devs_8234.append({"ip": "192.168.0.7", "port": 8234, "type": "fixed_8234"})
```

## 七、故障排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 继电器不响应 | 设备 IP 不是 .7 | 在设备 Web 管理页修改 IP 为 192.168.0.7 |
| 所有通道无效 | Unit ID 不匹配 | USR-IO808 默认 Unit=1，确认未被修改 |
| 高压不输出 | 需先开 `HighXZ` 再开 `HighKZ` | 按顺序操作：选择→主控 |
